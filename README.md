# 护龄智守

面向单房间固定机位场景的智能安全值守系统。系统持续接收视频输入，输出连续状态、风险提醒、过程留档与历史回看，用于家庭照护、护理值守和固定房间安全监看。

## 项目定位

护龄智守不是普通录像回看工具，也不是只看单帧的跌倒分类器。

它的核心目标是把视频流转成可以处理的照护过程：

| 用户关心的问题 | 系统给出的结果 |
| --- | --- |
| 当前是否安全 | 当前状态、风险强度、最近变化 |
| 是否需要马上去看 | 处置建议、正式提醒、最高风险时刻 |
| 之后怎么复盘 | 历史回看、过程摘要、关键时刻跳转 |

系统坚持一条主技术路线：

```text
RTMO -> Scene-Pose Temporal Net -> EventEngine -> FastAPI / Vue / SQLite / Docker
```

## 页面结构

| 页面 | 主要用途 |
| --- | --- |
| 实时值守 | 查看当前画面、状态、提醒、建议动作和过程时间线 |
| 历史回看 | 查看已经保存的过程，定位最高风险时刻和系统提醒 |
| 系统信息 | 查看接入方式、判断主链、运行参数和误报控制口径 |

历史回看只显示已经保存过的过程。实时值守页中的“保存到历史回看”会把当前正在看的过程写入历史记录，“开始新一段”会清空当前实时累计并重新判断。

## 模型主线

当前识别链路为：

```text
视频帧
  -> RTMO 姿态估计
  -> 骨架、运动学、场景关系特征
  -> Scene-Pose Temporal Net
  -> EventEngine
  -> 状态、提醒、回看记录
```

`Scene-Pose Temporal Net` 使用混合时序结构：

| 结构 | 作用 |
| --- | --- |
| Pose Encoder | 编码 17 点骨架与关键点置信度 |
| Quality-Aware Spatial Summary | 降低低质量关键点对帧表示的影响 |
| Feature Fusion | 融合骨架、运动学与场景关系特征 |
| Residual Temporal Blocks | 捕捉短时间局部动作变化 |
| Transformer Encoder | 建模更长时间范围内的动作过程 |
| Quality-Weighted Pooling | 对低质量帧降权后形成窗口级判断 |
| Prediction Heads | 输出帧级辅助分类、窗级分类和风险分数 |

当前运行时口径：

| 参数 | 当前值 |
| --- | --- |
| `window_size` | `64` |
| `inference_stride` | `4` |
| 状态类别 | `5` 类 |

## 版本状态

截至 `2026-04-11`：

| 方向 | 状态 |
| --- | --- |
| 应用系统 | 已具备 1.0 发布基础 |
| 输入链路 | 实时接入、模拟监看、上传复核均已打通 |
| 前端展示 | 已支持桌面端与移动端浏览 |
| 历史回看 | 已支持保存当前过程、异常状态筛选与关键时刻回放 |
| 模型实验 | `v21` 验证了中等阈值过渡标签和 recovery 采样加权，但晋级结论仍为 `hold` |

当前可以继续推进“系统 1.0”发布准备，但不把模型写成最终定版。`v21` 的样本级 `macro_f1` 达到 `0.7185`，`near_fall` 样本级 F1 达到 `0.6667`，但 `fall / recovery / prolonged_lying` 仍未通过晋级规则，后续模型重点仍是提升这些关键类别的稳定性。

## 快速启动

### Docker 运行

```bash
docker compose -f docker-compose.runtime.yml up --build
```

默认页面：

| 页面 | 地址 |
| --- | --- |
| 实时值守 | <http://127.0.0.1:18014/dashboard#/live> |
| 历史回看 | <http://127.0.0.1:18014/dashboard#/records> |
| 系统信息 | <http://127.0.0.1:18014/dashboard#/system> |

默认目录：

| 目录 | 内容 |
| --- | --- |
| `runtime-release/` | 运行时发布包 |
| `runtime-demo/` | 模拟监看视频、预测结果、会话报告 |
| `runtime-data/` | 运行时归档数据 |
