# 护龄智守状态定义手册

## 1. 文档目的

本手册用于统一系统内五类状态的业务定义、标注边界、运行时判读口径和阈值校准原则，避免出现“训练时一套标准、运行时一套标准、人工复核又是一套标准”的漂移问题。

本手册适用于以下环节：

- 公开视频与模拟监看视频的人工复核
- 区间标签回灌
- 运行时异常复查
- 阈值校准与误报分析

## 2. 状态集合

系统输出以下五类状态：

- `normal`：正常活动
- `near_fall`：高风险失衡
- `fall`：跌倒
- `recovery`：恢复起身
- `prolonged_lying`：长时间卧倒

## 3. 通用判读原则

### 3.1 以时间窗而不是单帧判读

系统不以单帧结果作为最终结论。状态必须结合连续时间窗内的骨架变化、风险分数变化和事件引擎输出共同判断。

### 3.2 以动作过程而不是最终姿态判读

仅凭“最后停在地上”不能直接判成 `fall`。必须观察是否存在明显姿态突变、重心快速下沉、躯干方向急剧改变和后续未及时恢复等过程特征。

### 3.3 低质量骨架不得强判

当运行时 `pose_quality_score`、`mean_keypoint_confidence`、`visible_joint_ratio` 同时偏低时，应优先视为“当前输入质量不足”，而不是直接提高风险等级。低质量骨架可以触发人工复核，但不应单独成为高风险事件依据。

## 4. 状态定义

### 4.1 `normal`

定义：
人员处于正常活动、正常坐卧、正常转身或稳定停留状态，没有形成明确失衡、跌倒或长卧风险。

典型情形：

- 正常行走
- 正常坐下和站起
- 正常弯腰、捡物、整理衣物
- 正常卧床、卧沙发
- 正常短时停留、缓慢转身

判读要点：

- 身体重心变化平滑
- 躯干方向变化连续
- 风险分数未持续升高
- 没有形成正式事件

### 4.2 `near_fall`

定义：
人员出现明显失衡、踉跄、异常下沉或短时失稳，但尚未形成已确认跌倒。

典型情形：

- 身体突然倾斜后又拉回
- 扶墙、扶桌、扶床边避免倒地
- 快速后撤、绊步、异常摇晃
- 接近地面但随后恢复平衡

判读要点：

- 必须出现明确的风险上升过程
- 持续时间通常短于正式跌倒后的滞留
- 若随后形成倒地并未恢复，应升级为 `fall`
- 若短时间内恢复稳定，可回落到 `recovery` 或 `normal`

### 4.3 `fall`

定义：
人员发生明确跌倒，表现为较短时间内重心快速下沉、姿态突变，并在地面或低位区域形成倒地结果。

典型情形：

- 前向、侧向、后向倒地
- 滑倒后快速失去站立姿态
- 坐位或半蹲位突然失控倒地

判读要点：

- 必须存在明显动作突变
- 躯干方向、髋部高度、关节展开形态出现显著变化
- 风险分数通常伴随快速抬升
- 仅有“缓慢坐地”或“正常躺下”不能直接判为 `fall`

### 4.4 `recovery`

定义：
人员从高风险失衡、跌倒或卧倒状态中恢复，出现明确起身、回正或重新获得稳定控制的过程。

典型情形：

- 倒地后自行起身
- 失衡后通过扶物恢复稳定
- 长卧后逐步坐起并恢复正常姿态

判读要点：

- 必须出现在 `near_fall`、`fall` 或 `prolonged_lying` 之后
- 不能把所有风险下降都当成 `recovery`
- 应体现“由低位向高位”的连续恢复过程

### 4.5 `prolonged_lying`

定义：
人员处于卧倒或低位停留状态，并且持续时间超过配置阈值，形成需要重点处理的长时间卧倒事件。

典型情形：

- 跌倒后持续倒地不起
- 长时间侧卧或俯卧在地面区域
- 倒地后无明显恢复动作

判读要点：

- 需要结合持续时间阈值
- 需要结合地面或低位区域关系
- 正常卧床、正常躺沙发不得判入此类

## 5. 高混淆场景处理原则

### 5.1 正常躺卧 vs `prolonged_lying`

必须结合场景位置和先验区域判断。位于床、沙发等允许卧姿区域的稳定躺卧优先保持 `normal`。

### 5.2 缓慢坐地 vs `fall`

若动作过程平缓、风险分数未显著抬升、后续存在稳定坐姿，应优先视为 `normal` 或 `near_fall`，而不是 `fall`。

### 5.3 姿态缺失 vs 高风险

被遮挡、低光、被子覆盖等导致的关键点置信度下降，不应直接升格为高风险。必须优先查看运行时姿态质量指标。

## 6. 姿态质量观测口径

系统运行时输出以下三项质量指标：

- `pose_quality_score`
  窗口级综合骨架质量分数，范围为 0 到 1。
- `mean_keypoint_confidence`
  当前时间窗内全部关键点的平均置信度。
- `visible_joint_ratio`
  当前时间窗内置信度达到可见阈值的关节占比。

建议口径：

- `pose_quality_score < 0.35`
  视为骨架质量偏低，应谨慎解读风险结果。
- `mean_keypoint_confidence < 0.30`
  说明当前关键点整体不稳定。
- `visible_joint_ratio < 0.40`
  说明当前可用关节过少，易出现误判。

以上阈值用于复核提示，不作为单独事件阈值。

## 7. 阈值校准原则

### 7.1 校准数据要求

事件阈值不得直接依据训练集或验证集调参结果。阈值校准必须使用独立的视频集合，且应尽量接近目标部署条件。

### 7.2 校准记录要求

每次调整以下阈值时，必须记录来源视频和调整依据：

- `near_fall_threshold`
- `fall_threshold`
- `recovery_threshold`
- `prolonged_lying_threshold`
- `prolonged_lying_seconds`

### 7.3 应用侧优先级

对于居家和照护场景，阈值调整优先级为：

1. 先压明显误报
2. 再提升真实跌倒召回
3. 最后再收紧边缘状态的过度波动

## 8. 复核与回灌要求

- 复核队列必须保留原始时间戳和状态上下文
- 区间标签回灌时必须注明来源为“人工复核”
- 对 `near_fall` 和 `recovery` 的回灌应特别谨慎，避免把主观语义判断硬灌成训练真值

## 9. 维护原则

- 不新增第二套状态体系
- 不在训练链和运行时链使用不同状态语义
- 不通过修改界面文案掩盖模型误判
- 所有状态定义调整都必须同步更新本手册
