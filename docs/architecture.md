# 系统架构

## 1. 架构目标

护龄智守采用单一路线架构，目标是在固定摄像头场景下提供稳定、可部署、可追溯的安全值守能力。系统同时满足实时推理、会话归档、历史回看和后续再训练需要。

## 2. 架构分层

### 2.1 房间先验层

房间先验层在部署前执行一次，用于生成固定房间的区域先验。当前采用：

- `Grounding DINO`
- `SAM 2`

输出内容包括床、沙发、地面活动区域和风险边缘。该层不进入实时主链。

### 2.2 姿态感知层

姿态感知层采用 `RTMO`，负责逐帧输出人体骨架和关键点置信度。该层为后续时序模型提供稳定输入。

### 2.3 时序识别层

时序识别层采用 `Scene-Pose Temporal Net`，输入包括：

- 标准化骨架序列
- 关键点置信度
- 运动学特征
- 场景关系特征

输出状态包括：

- `normal`
- `near_fall`
- `fall`
- `recovery`
- `prolonged_lying`

### 2.4 事件层

事件层采用 `EventEngine`，将连续状态流稳定为事件流。该层输出近跌倒提示、跌倒、长时间卧倒和恢复事件。

### 2.5 应用层

应用层由以下模块组成：

- `FastAPI` 运行时服务
- 运行时前端面板
- `SQLite` 历史会话归档
- Docker 部署封装

## 3. 训练架构

训练主线如下：

1. 生成原始视频 manifest
2. 执行离线骨架抽取
3. 生成特征缓存
4. 构建时间窗
5. 训练模型
6. 进行样本级、事件级和应用级评估
7. 打包发布并执行批量验证

训练过程中强制校验 `raw manifest -> pose manifest` 覆盖率，避免残缺样本进入训练。

## 4. 运行时架构

运行时链路如下：

1. 视频流输入
2. `RTMO` 骨架估计
3. 特征整理
4. `Scene-Pose Temporal Net` 推理
5. `EventEngine` 生成事件
6. FastAPI 输出接口结果
7. 前端展示与 `SQLite` 归档

## 5. 不采用的方案

- 不采用 `OpenPose` 作为主线
- 不采用纯几何阈值规则法作为主线
- 不引入第二套模型架构
- 不提供双运行时切换
- 不将大规模分割模型放入实时链路
